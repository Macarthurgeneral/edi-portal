EDI Reference
// ANSI X12 transaction set documentation
850
Purchase Order
Sent by OrderDog POS to your portal. Contains items to be ordered, quantities, pricing, delivery date, and shipping terms.
DIRECTION: OrderDog → You
997
Functional Acknowledgement
Confirms receipt and syntactic validity of any inbound EDI transaction. Must be sent back to OrderDog after receiving an 850.
DIRECTION: You → OrderDog
855
Purchase Order Acknowledgement
Responds to the 850 with acceptance, rejection, or modification. Tells OrderDog whether you can fulfill the order as requested.
DIRECTION: You → OrderDog
810
Invoice
Electronic invoice submitted to OrderDog after fulfillment. Replaces paper invoices. Use the Invoice Converter tool to auto-generate from PDF/CSV.
DIRECTION: You → OrderDog
Catalog Flat File (Daily FTP)
OrderDog picks up a daily flat file from this server. The file should be a pipe-delimited (|) or tab-delimited .txt / .csv with the following recommended columns:

UPC | ITEM_DESC | PACK_SIZE | UNIT_COST | RETAIL_PRICE | CATEGORY | VENDOR_CODE | ACTIVE 012345678901 | PEPSI 20OZ | 24 | 14.40 | 1.99 | BEVERAGES | PEPSI | 1
Upload your catalog file to the Catalog Flat Files section. OrderDog polls this portal daily to pick it up.
